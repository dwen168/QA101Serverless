How it works

High-level only, not a formal spec
STEP 1 · DATA INGESTION
Collects price action, valuation, news sentiment, and macro anchors (CL, TNX, VIX).
STEP 2 · SIGNAL BUILD
Builds technical + fundamental + sentiment features and standardizes them into comparable signal scores.
STEP 3 · RISK OVERLAY
Applies volatility and macro penalties, conflict checks, and confidence calibration to avoid over-aggressive calls.
STEP 4 · FINAL SCORING
Combines weighted factors into a composite score and maps it into Buy / Hold / Sell thresholds.
STEP 5 · RECOMMENDATION
Outputs recommendation + confidence + rationale summary so users can inspect why the decision was made.

Recommendation Algorithm Specification

Implementation-oriented view of how the current recommendation engine is built
A. SCORING EQUATION
Implemented in scoreSignals()
adjustedPoints_i = roundToHalf(rawWeight_i × multiplier_bucket,horizon)
compositeScore = Σ adjustedPoints_i
Each signal contributes signed points. Raw weights come from the calibrated weight file. The selected investment horizon then rescales whole signal buckets before the final sum is formed.

D. OVERLAYS & ANTI-DOUBLE COUNTING
  • Unified CPI: Consolidates RBA and ABS CPI inflation. If CPI > 3%, it triggers Macro CPI Inflation Pressure and neutralizes RBA bias from policy headwinds.
  • MA Guard: Skips EDA Trend Strength if the primary Price > MA50 signal has already triggered.
  • MACD Decoupling: RSI boundaries adapt purely on MA alignment and EDA volatility; MACD only scores independently.
  • OBV Volume: Volume confirmation relies solely on OBV; redundant EDA Volume metrics are removed.
  • Breakout Guard: Active EDA Breakout suppresses short-term Daily Momentum to prevent single-candle over-scoring.
  • Event Regime & Anchors: Adds sector-aware bias for war/oil regimes, validated against oil (CL=F), yields (^TNX), and VIX.
E. ACTION MAPPING
The composite score is mapped into a recommendation band after all adjusted signals are summed.
SHORT/MEDIUM: score >= 7 -> STRONG BUY, score >= 4 -> BUY, score <= -4 -> SELL, score <= -7 -> STRONG SELL
LONG: score >= 8 -> STRONG BUY, score >= 5 -> BUY, score <= -5 -> SELL, score <= -8 -> STRONG SELL
-2 <= score <= 2 -> HOLD
else -> HOLD
Mapping is centralized in shared profiles (getActionThresholds/mapActionFromScore) and reused by both recommendation and backtest. The UI then shows grouped attribution plus a separate bearish-pressure summary (not double-counted as another pillar).
F. RISK OUTPUTS
ATR14 → stopLoss, takeProfit
VaR95 → 1-day downside reference
confidence → score strength + signal agreement - conflict drag
Recommendation output contains actionable levels, risk/reward, confidence, and explanation. The ATR parameters are horizon-specific.
F2. CONFIDENCE EQUATION
Implemented in computeConfidence()
normalizedScore = min(1, abs(score) / 10)
baseConfidence = 42 + round(34 * tanh(normalizedScore * 1.8))
alignment = abs(posMag - negMag) / (posMag + negMag)
consistencyAdjustment = round((alignment - 0.5) * 14)
Confidence starts from score magnitude, then gets adjusted by how one-sided the evidence is. Strongly mixed positive and negative evidence reduces conviction even when the raw score is non-zero.
F3. CONFIDENCE ADJUSTMENTS
Final confidence is clipped to 30-92
if alignment < 0.3 and totalMagnitude >= 5: -6 conflict penalty
if signalCount <= 3: -4
if signalCount >= 10: +2
if macroRisk == HIGH: -8
if macroRisk == LOW: +3
finalConfidence = clamp(rawConfidence, 30, 92)
This means confidence is not just a monotonic function of score. It also reacts to conflict, evidence depth, and macro regime quality.
G. ATR EXIT SPEC
Used by recommendation output and aligned with backtesting
stopLoss = entry - ATR14 × SLmultiplier(horizon)
takeProfit = entry + ATR14 × TPmultiplier(horizon)
SHORT: SL = entry - 1.2 × ATR14, TP = entry + 2.0 × ATR14
MEDIUM: SL = entry - 1.5 × ATR14, TP = entry + 2.5 × ATR14
LONG: SL = entry - 2.0 × ATR14, TP = entry + 4.0 × ATR14
H. MODULE BOUNDARIES
What is and is not a backtest
Historical Pattern Analogs is not a backtest. It scans for past RSI-zone and MA50-position analogs and reports forward 5-day / 10-day outcomes. It is context, not trade simulation.
The backtest strategy named trade-recommendation reuses the shared price-and-technical signal core, but it does not replay historical sentiment, macro headlines, analyst revisions, or fundamental release timing.
